# Company PPT Style Guide (for Claude Code / Codex)

이 레포는 **[이노션] 전용 PPT 디자인 스타일 가이드**입니다.  
Claude Code, GitHub Copilot, Codex 등 AI 에이전트가 이 레포의 규칙과 예시를 참고해서, 일관된 디자인의 슬라이드를 생성할 수 있도록 설계되었습니다.

---

## 1. 목적

- 우리 회사 발표 자료의 **디자인 일관성 유지**
- 디자이너가 없어도 **기본 이상 퀄리티** 확보
- Claude Code / Codex가 참조할 수 있는 **텍스트 기반 스타일 가이드** 제공

---

## 2. 구조

- `design-tokens/tokens.json`  
  → 색상, 폰트, 여백, 카드 스타일 등 공통 디자인 토큰 정의

- `templates/`  
  → 슬라이드 패턴별 템플릿 (표지, 아젠다, 2컬럼, 3카드, KPI, 차트, 마무리 등)

- `examples/`  
  - `good/` : 벤치마크할 만한 좋은 슬라이드 예시 (외부/내부) + 코멘트  
  - `bad/`  : 피해야 할 슬라이드 예시 + 왜 나쁜지 설명

- `assets/`  
  → 로고, 아이콘, 배경 이미지 등 재사용 자산

- `prompts/`  
  → Claude Code / Codex에서 이 레포를 참고해 PPT를 만들 때 사용하는 프롬프트 예시

---

## 3. 디자인 토큰 (요약)

자세한 값은 `design-tokens/tokens.json` 참고.

- 컬러
  - 기본 배경: 다크/라이트 중 하나 고정
  - 포인트 컬러: 1~2개만 사용 (예: 메인 컬러 + 서브 컬러)
- 폰트
  - 제목: [예: Pretendard Bold, 28~36pt]
  - 본문: [예: Pretendard Regular, 16~18pt]
- 레이아웃 룰
  - 한 슬라이드 **핵심 메시지 1개**
  - 글머리표: 최대 4~5 줄
  - 카드: 한 줄에 최대 3개, 설명은 2~3줄

---

## 4. 슬라이드 템플릿 규칙

각 템플릿 상세는 `templates/*.md` 참고.

- Cover (표지)
  - 회사 로고 + 제목 + 부제
  - 배경은 [다크/라이트] 톤, 이미지 과하지 않게
- Agenda
  - 3~5개 섹션으로만 구성
- Problem / Solution
  - 왼쪽: 문제 요약, 오른쪽: 해결책 요약
- 3 Cards (Feature, Benefit 등)
  - 한 줄에 3개 카드, 각 카드 제목 + 2줄 설명
- KPI / Numbers
  - 숫자 3~4개만, 카드나 큰 타이포로 강조
- Closing
  - 한 줄 메시지 + 연락처/QR 코드

---

## 5. 좋은/나쁜 예시 정리 방식

`examples/good`와 `examples/bad`는 실제 PPT 파일 대신, **슬라이드 구조를 텍스트로 정리**합니다.

예시:

```text
Title: 2025 상반기 캠페인 성과

- 왜 좋은가?
  - 제목이 짧고 핵심이 명확함
  - KPI 카드 3개, 각 카드 2줄 이내
  - 색상 2개만 사용 (브랜드 컬러 + 회색)
  - 불필요한 아이콘/이미지 없음
```

이렇게 “왜 좋은지/나쁜지”를 설명해두면, AI가 스타일을 학습하는 데 도움이 됩니다.

---

## 6. Claude Code / Codex 사용 가이드 (요약)

자세한 프롬프트는 `prompts/` 폴더 참고.

예시:

> 이 GitHub 레포의 내용을 스타일 가이드로 사용해서,  
> [목적: 예. B2B SaaS 서비스 소개용] 10장짜리 프레젠테이션을 만들어줘.  
> `design-tokens/tokens.json`의 색상/폰트/레이아웃 규칙을 지키고,  
> `templates/` 폴더의 구조를 최대한 따라줘.
